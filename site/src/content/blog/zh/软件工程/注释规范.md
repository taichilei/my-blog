# 注释规范

在实际开发中，这些特殊注释的使用频率大致可以按照以下顺序排列（从高到低）：

1. TODO（使用最频繁）

​	•说明需要完成的任务，广泛用于团队协作和个人开发。

​	•例如：# TODO: 实现用户登录逻辑

2. FIXME

​	•表示代码有问题需要修复，通常在发现 bug 但暂时没时间修复时使用。

​	•例如：# FIXME: 这里可能会导致索引越界

3. DEBUG

​	•调试过程中用于标记暂时注释掉的代码，避免误删有用的调试信息。

​	•例如：# DEBUG: 记录 API 响应数据

4. NOTE

​	•说明代码的特殊情况、边界条件或者特殊逻辑，帮助理解。

​	•例如：# NOTE: 这个函数假设输入已排序

5. HACK

​	•表示临时解决方案，代码可能不优雅但能工作，提醒后续优化。

​	•例如：# HACK: 用 sleep 解决竞争问题，后续应使用锁

6. BUG

​	•明确标记代码中已知的 bug，便于后续修复。

​	•例如：# BUG: 负数输入时会导致崩溃

7. OPTIMIZE

​	•提醒后续优化代码，提高性能或简化逻辑。

​	•例如：# OPTIMIZE: 这里可以用多线程加速

8. XXX

​	•代表代码中不确定或危险的部分，提醒开发者注意。

​	•例如：# XXX: 这个边界条件可能有问题，需要测试

9.TEMP

临时注释掉某些代码，可能稍后恢复。

​	•例如：# TEMP: 先注释掉日志输出，避免干扰测试
